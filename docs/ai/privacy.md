# AI Privacy, Prompt Design & Clinical Boundaries

---

## Privacy Architecture

| Data type | Storage | Sent to AI | User control |
|---|---|---|---|
| Journal text | Supabase (encrypted at rest) | Aggregated sentiment only (unless opted in) | Full export + delete |
| Mood / cycle logs | Supabase | Aggregated trends | Full export + delete |
| Sleep / wearable data | Supabase | Aggregated scores | Disconnect at any time |
| Medication adherence | Supabase | Structured tags only | Off by default for psychiatrist sharing |
| Social rhythm anchors | Supabase | Consistency score | Full export + delete |
| Community posts | Supabase | Full text (for moderation only) | Delete at any time |
| Relapse Signature | Supabase (encrypted) | Included in report prompt (structured) | Edit / delete at any time |
| Life events | Supabase | Event type + date only | Full export + delete |

**Zero retention:** Groq's API does not log or store inference inputs or outputs. No user data is used for model training.

**No third-party analytics:** No Firebase Analytics, Amplitude, Mixpanel, or equivalent SDKs. Telemetry is internal only.

**Immediate deletion:** Users can request full account and data deletion at any time. Deletion is executed within 24 hours with confirmation.

## Prompt Engineering

Prompts are designed to work entirely with structured, aggregated data — never raw PII. The payload schema enforces this at the aggregation layer before any call is made.

### Wellness Report Prompt

```
[System]
You are a clinical support AI for Equi, a bipolar disorder monitoring app.
You help generate weekly wellness summaries for users to share with their psychiatrists.
You do not diagnose. You surface patterns and correlations for clinical interpretation.
Use clear, warm, non-alarmist language. Never label mood states as "bad" or "good".

[Context block — structured, no raw PII]
Reporting period: {start_date} to {end_date}

Sleep data:
- Average: {avg_sleep}h
- Nights below 5h: {low_sleep_nights}
- Trend: {sleep_trend}

Cycle states:
- {cycle_state_summary}

Rhythm Consistency Score:
- This week: {rhythm_score}/100
- Previous week: {prev_rhythm_score}/100

Medication adherence:
- Adherence rate: {adherence_pct}%
- Missed doses: {missed_count}
- Side effects flagged: {side_effect_tags}

Life events this period:
- {life_events_list}

Relapse Signature — Manic early signs:
- {manic_signatures}

Relapse Signature — Depressive early signs:
- {depressive_signatures}

Journal sentiment trend:
- {sentiment_summary}

[Task]
Generate a wellness report with the following sections:
1. Period Overview (2–3 sentences)
2. Sleep & Rhythm
3. Mood & Cycle
4. Medication Adherence
5. Life Events Impact
6. Early Warning Flags (cross-reference against Relapse Signature)
7. What to discuss with your psychiatrist
```

Always assume good intent unless clearly otherwise.

## AI Limitations and Clinical Boundaries

Equi's AI is explicitly scoped to pattern surfacing — not diagnosis, not treatment recommendation.

**What the AI does:**
- Surface correlations between tracked data points
- Flag when patterns match a user's own Relapse Signature
- Generate structured summaries for psychiatrist review
- Pre-moderate community content for safety

**What the AI does not do:**
- Diagnose bipolar disorder or any condition
- Recommend medication changes
- Replace clinical judgement
- Provide crisis counselling (crisis mode routes to human contacts and lines)

Every AI-generated report includes a plain-language disclaimer visible before sharing:

> *"This report was generated by AI to support your appointment — it is not a clinical assessment. Your psychiatrist makes all clinical decisions."*
