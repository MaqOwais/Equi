# Phase 4D — PDF Export & Data Sharing

Generates a clinical-quality PDF from the AI Wellness Report and allows the user to share it with their psychiatrist, a well-wisher/guardian, or save it locally. Also covers the full personal data export (GDPR/CCPA compliance path).

## PDF Report

The exported PDF is a clean, readable version of the AI Wellness Report screen. It is generated server-side by a Supabase Edge Function to avoid bundling PDF libraries in the mobile app.

### What the PDF contains

1. **Cover** — Equi logo, user's first name, report period, generated date.
2. **Summary** — AI-generated plain-language summary paragraph.
3. **Cycle Overview** — 7-day state table (coloured cells: green/blue/purple/amber).
4. **Mood Chart** — bar chart of daily mood scores.
5. **Sleep** — nightly duration bar chart with cycle-state colour overlay (if sleep data available).
6. **Social Rhythm** — 7-day score line (if social rhythm data available).
7. **Medication Adherence** — taken/skipped/partial per day (if medication tracking on).
8. **Substances** — alcohol/cannabis day counts.
9. **Activities Completed** — list of activity names.
10. **Early Warning Flags** — if any, displayed in amber with disclaimer.
11. **Nutrition Snapshot** — if ≥ 3 days of nutrition data in the period.
12. **Privacy Footer** — "Generated by Equi. Not a clinical diagnosis. Zero AI data retention."

### What the PDF never contains

- Raw journal text
- Community posts
- Psychiatrist contact details
- Any data from other users
- Precise location data

## PDF Generation — Edge Function

**Function name:** `generate-report-pdf`
**Trigger:** POST from mobile app with `{ report_id: string, user_id: string }`

### Technology choices

| Option | Verdict |
|---|---|
| **Puppeteer (headless Chromium)** | Best quality, renders HTML/CSS faithfully. Heavy (~300MB). Suitable for Edge Function. |
| **jsPDF + autoTable** | Lighter, no headless browser, but limited layout control. |
| **PDFKit** | Node.js, good for programmatic layouts. No HTML rendering. |

**Recommended:** Puppeteer via `@sparticuz/chromium` (lightweight Chromium for serverless).

### Flow

```
Mobile app
    │  POST /generate-report-pdf { report_id }
    ▼
Edge Function
    ├── Fetch report row from ai_reports (verify user_id matches auth.uid())
    ├── Build HTML string from report_json (inline CSS, no external assets)
    ├── Launch Chromium → print to PDF (A4, print media)
    ├── Upload PDF to Supabase Storage: reports/{user_id}/{report_id}.pdf
    ├── Create signed URL (7-day expiry for share links, 1h for direct download)
    └── Return { url: string, expires_at: string }
Mobile app
    ├── Download → share via expo-sharing (saves to Files / triggers share sheet)
    └── Share with contact → POST /notify-companion { companion_id, pdf_url }
```

### Edge Function skeleton

```ts
// supabase/functions/generate-report-pdf/index.ts
import chromium from '@sparticuz/chromium';
import puppeteer from 'puppeteer-core';

Deno.serve(async (req) => {
  const { report_id } = await req.json();
  const authHeader = req.headers.get('Authorization')!;

  // Verify user owns this report
  const { data: report } = await supabase
    .from('ai_reports')
    .select('*')
    .eq('id', report_id)
    .single();

  // Build HTML
  const html = buildReportHtml(report.report_json, report.period_start, report.period_end);

  // Render PDF
  const browser = await puppeteer.launch({
    args: chromium.args,
    executablePath: await chromium.executablePath(),
    headless: true,
  });
  const page = await browser.newPage();
  await page.setContent(html, { waitUntil: 'networkidle0' });
  const pdfBuffer = await page.pdf({ format: 'A4', printBackground: true });
  await browser.close();

  // Upload to Storage
  const path = `${report.user_id}/${report_id}.pdf`;
  await supabase.storage.from('reports').upload(path, pdfBuffer, {
    contentType: 'application/pdf',
    upsert: true,
  });

  // Return signed URL (1h for direct download)
  const { data: { signedUrl } } = await supabase.storage
    .from('reports')
    .createSignedUrl(path, 3600);

  return new Response(JSON.stringify({ url: signedUrl }), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

## Companion Sharing

When a user shares a PDF with a companion (psychiatrist or well-wisher):

1. A 7-day signed URL is generated.
2. A row is inserted into a new `report_shares` table.
3. An email is sent to the companion via Supabase's built-in email (or Resend).
4. The companion receives a read-only link — no Equi account required.

### `report_shares` table

```sql
create table report_shares (
  id            uuid default gen_random_uuid() primary key,
  report_id     uuid references ai_reports on delete cascade not null,
  user_id       uuid references auth.users on delete cascade not null,
  companion_id  uuid references companions on delete set null,
  share_url     text not null,
  expires_at    timestamptz not null,
  viewed_at     timestamptz,
  created_at    timestamptz default now()
);

alter table report_shares enable row level security;
create policy "Users manage their own shares"
  on report_shares for all using (auth.uid() = user_id);
```

## Personal Data Export (GDPR / CCPA)

Separate from the AI report export. A full data dump in JSON + CSV format.

**Edge Function:** `export-user-data`

### Contents of the export ZIP

```
equi-export-{date}/
├── profile.json            — display_name, diagnosis, preferences
├── mood_logs.csv           — date, score
├── cycle_logs.csv          — date, state, intensity, symptoms
├── journal_entries.csv     — date, content (full text, unredacted)
├── medication_logs.csv     — date, status, skip_reason
├── sleep_logs.csv          — date, duration, quality, source
├── social_rhythm_logs.csv  — date, score, anchors_hit
├── nutrition_logs.csv      — date, categories
├── activity_completions.csv — date, activity_name, note
├── ai_reports.json         — all generated reports (JSON)
├── relapse_signatures.json — manic + depressive signatures
└── README.txt              — explains each file, data provenance
```

**Trigger:** `You → Settings → Privacy & data → Export my data`
**Delivery:** Signed URL (24h) emailed to the user's registered email.

### Deletion

`You → Settings → Privacy & data → Delete my account`:
1. Cascade deletes all user rows (auth + all tables via `on delete cascade`).
2. Supabase Storage files purged via Edge Function.
3. Confirmation email sent.
4. 30-day soft-delete grace period before permanent deletion (stored as `deletion_scheduled_at` on profile).
