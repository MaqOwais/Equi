# Access: Well-wisher

A well-wisher is someone the user has chosen to keep informed about their wellbeing — a partner, sibling, therapist, or anyone the user trusts. They download the Equi app, choose "I'm supporting someone" at the role selection screen, and get a companion home view. They can see what the user shares and send messages and check-ins back — the experience is two-way.

Managed from **Screen 16 — Well-wisher & Guardian Access**.

---

## What a Well-wisher Can Access

All permissions below are **off by default** and set individually per person. The user can turn any permission on or off at any time.

| Data Type | Access | Default | What They See |
|---|---|---|---|
| Mood summaries | 🔓 Toggle | Off | A simplified daily summary — cycle state and mood level, not raw scores |
| Selected journal entries | 🔓 User selects | Off | Only entries the user has explicitly marked as shareable; full text visible |
| AI Wellness Report | 🔓 Manual share | Off | User taps "Share with [name]" from Screen 09; the well-wisher receives a read-only PDF |
| High-risk threshold alerts | 🔓 Toggle | Off | Push notification if mood drops below a threshold for 2+ consecutive days (threshold is user-configured) |

### Never accessible

| Data Type | Status |
|---|---|
| Raw mood scores (numeric) | ❌ Never — summaries only |
| Full journal (unsolicited) | ❌ Never — user must select individual entries |
| Medication adherence | ❌ Never |
| Substance use data | ❌ Never |
| Sleep / wearable data | ❌ Never — only appears in AI report if shared |
| Relapse signatures | ❌ Never |
| Bipolar Workbook | ❌ Never |
| Prescribed activities | ❌ Never |
| Activity compliance | ❌ Never |
| Community posts | ❌ Never |
| Crisis history | ❌ Never |
| Emergency contacts | ❌ Never |
| Account management | ❌ Never — well-wishers cannot take action on the account |

## High-Risk Alerts

When enabled, the well-wisher receives a push notification if the user's mood log falls below the configured threshold for a sustained period. The alert message is non-clinical and non-alarming:

> *"[User] has shared that they're having a difficult few days. You might want to reach out."*

The alert does not include mood scores, journal content, or any clinical detail. The user configures the threshold and the wording from Screen 16.

---

## What a Well-wisher Can Send

From the companion app, well-wishers can reach out without needing to know what to say:

- **Quick check-ins** — one-tap templates: "Thinking of you 💜", "How are you?", "Proud of you 🌟"
- **Direct messages** — private text messages, visible only to the well-wisher and the user
- **Reactions to shared journal entries** — 💜 or 🙏 (no free-text reactions, to avoid adding pressure)

The user receives these in their Equi app. They can reply, ignore, or mute the well-wisher at any time.

## What a Well-wisher Cannot Do

- They cannot log anything on behalf of the user
- They cannot see what other well-wishers have access to
- They cannot change any permissions — only the user can
- They cannot request additional access

---

## How Sharing Works

The user adds a well-wisher from Screen 16 — entering their name and relationship. An invite is sent to the well-wisher's email or phone. Once they accept, the user sets permissions for them individually. The user can:
- Adjust any permission at any time
- Remove the person entirely (immediately revokes all access)
- Share a single AI report with them one-time without granting ongoing access

The well-wisher never sees what permissions have been declined — they only see what has been shared with them.
